### 2. PMBOK6核心公式不完整

#### 2.1 EVM公式缺失TCPI ❌
**影响文件**: 02-过程域核心考点精解.md, 05-高频工具技术与模型汇总.md

**当前EVM公式**:
- PV, EV, AC, BAC ✅
- CV = EV - AC ✅
- SV = EV - PV ✅
- CPI = EV / AC ✅
- SPI = EV / PV ✅
- EAC = BAC / CPI ✅
- ETC = EAC - AC ✅
- VAC = BAC - EAC ✅

**缺失的关键公式**:
- **TCPI = (BAC - EV) / (BAC - AC)** [按BAC]
- **TCPI = (BAC - EV) / (EAC - AC)** [按EAC]

**TCPI说明**:
- 含义: 完工绩效指数，为了实现特定管理目标，剩余资源使用必须达到的成本绩效指数
- 重要性: EVM体系的重要补充，用于判断剩余工作的绩效要求
- 解读: TCPI > 1表示需要比当前更高效，TCPI < 1表示可以比当前效率低

**补充要求**:
- 公式定义和含义
- 计算示例(带数据)
- 考试应用场景
- 与CPI的关系说明

**重要性**: ⭐⭐⭐⭐⭐ (高频计算题)

**影响文件**: 02-过程域核心考点精解.md, 05-高频工具技术与模型汇总.md

**当前公式**:
- 期望值 = (乐观 + 4×最可能 + 悲观) / 6 ✅

**缺失公式**:
- **标准差(σ) = (悲观 - 乐观) / 6**

**缺失的正态分布应用**:

### 4. 混合方法内容不足

**影响文件**: 04-敏捷与混合方法深度解析.md

**当前状态**: 仅简单提及"硬件+软件"使用混合方法

**需要补充的4种混合模式**:
1. **预测型主导，敏捷补充**: 整体瀑布框架，部分模块用敏捷
2. **敏捷主导，预测型补充**: 敏捷开发，但合规/采购用预测型
3. **并行混合**: 不同子项目/团队用不同方法
4. **阶段混合**: 早期探索用敏捷，后期稳定后用预测型

**典型混合场景**:
- 需要监管合规的创新项目
- 大型企业数字化转型项目
- 涉及硬件+软件的产品开发
- 固定合同但需求不确定的项目
- 分布式团队协作项目

**混合方法的挑战**:
- 不同方法的术语和文化冲突
- 治理和报告的复杂性
- 团队技能和适应能力
- 干系人期望管理

**重要性**: ⭐⭐⭐⭐⭐ (混合方法题目占比增加)
