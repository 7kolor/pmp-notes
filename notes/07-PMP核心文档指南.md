# 07-PMP核心文档指南 (扩充版)

> 本篇笔记是您PMP备考的“**文档说明书**”。PMP考试中充斥着各种名称相似的“计划”、“基准”、“登记册”等。本篇旨在用“考点”形式，按照十大知识领域，帮您彻底厘清这些核心文档的用途，让您在情景题中能瞬间定位到正确的文件。

---

## 核心概念：计划、基准与项目文件

在深入了解具体文档前，我们先建立一个框架：

- **管理计划 (Management Plan)**: 回答“**How? / 如何做？**”的问题。是指导方针和游戏规则。例如，《范围管理计划》规定了“如何定义范围”。
- **基准 (Baseline)**: 是经正式批准的、用于与实际绩效进行比较的“**官方测量尺**”。PMP有三大基准：**范围基准、进度基准、成本基准**。它们是项目管理计划的一部分，但非常特殊，只有通过正式的变更控制流程才能修改。
- **项目文件 (Project Documents)**: 除计划和基准外的所有其他文件，是项目过程中的记录和产出。例如：风险登记册、问题日志等。它们是动态的，会随着项目进展而持续更新。

---

## 按知识领域划分的核心文档

### 4. 整合管理 (Integration Management)
> 整合管理是项目经理的专属领域，其文档是贯穿项目始终的“顶层文件”。

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **项目章程 (Project Charter)** | 项目的“**出生证明**”+<br>项目经理的“**授权委托书**” | **正式启动项目**，授权PM使用资源，明确总体目标和关键干系人。PM被任命后，首要任务就是确保有已批准的章程。 |
| **项目管理计划 (Project Management Plan)** | 项目的“**总作战地图**” | 整合所有子计划和基准，是指导项目执行、监控和收尾的**总纲**。 |
| **假设日志 (Assumption Log)** | “**丑话说在前面**”的清单 | 记录项目规划所依据的所有假设条件和制约因素。有助于风险识别。 |
| **变更日志 (Change Log)** | 项目的“**变更流水账**” | 记录所有变更请求的提交、分析、批准或否决的状态。用于跟踪和沟通变更。 |
| **问题日志 (Issue Log)** | 项目的“**待办问题清单**” | 记录和跟踪项目中出现的所有问题，确保它们得到解决，不会被遗忘。 |
| **经验教训登记册 (Lessons Learned Register)** | 项目的“**成长日记/错题本**” | 在**项目全周期中持续记录**从项目中获得的知识和教训。不仅仅是收尾时才做。 |
| **最终报告 (Final Report)** | 项目的“**毕业总结**” | 在项目收尾时编制，总结项目整体绩效，包括范围、进度、成本、质量等目标的达成情况。 |

### 5. 范围管理 (Scope Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **范围管理计划** | 如何管范围的“**说明书**” | 规定了如何规划、定义、确认和控制项目范围。 |
| **需求管理计划** | 如何管需求的“**说明书**” | 规定了如何分析、记录和管理项目需求。 |
| **范围基准 (Scope Baseline)** | 范围的“**官方测量尺**” | 由“**项目范围说明书、WBS、WBS词典**”三部分组成。用于衡量范围绩效，是变更控制的依据。 |
| **需求文件 (Requirements Documentation)** | “**客户想要什么**”的清单 | 详细描述了所有干系人的需求，是范围和产品设计的基础。 |
| **需求跟踪矩阵 (RTM)** | 需求与成果的“**亲子鉴定**” | 将每一项需求与它对应的业务目标、WBS、测试用例等链接起来，确保需求不被遗漏。 |

### 6. 进度管理 (Schedule Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **进度管理计划** | 如何管进度的“**说明书**” | 规定了如何制定、管理、执行和控制项目进度。 |
| **进度基准 (Schedule Baseline)** | 进度的“**官方测量尺**” | 经过批准的项目进度计划版本。用于与实际进度对比，计算进度偏差(SV)。 |
| **项目进度计划 (Project Schedule)** | 项目的“**详细时间表**” | 包含了所有活动、持续时间、依赖关系和计划日期的动态日历。甘特图是其最常见的表现形式。 |
| **活动清单 (Activity List)** | “**要做哪些事**”的清单 | WBS工作包的进一步分解，是制定进度的基础。 |
| **里程碑清单 (Milestone List)** | 项目的“**关键节点**”清单 | 列出了项目中的重要时间点或事件，用于向高层沟通进度。 |

### 7. 成本管理 (Cost Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **成本管理计划** | 如何管成本的“**说明书**” | 规定了如何规划、估算、预算和控制项目成本。 |
| **成本基准 (Cost Baseline)** | 成本的“**官方测量尺**” | 经过批准的、按时间段分配的项目预算（呈S曲线）。用于与实际成本对比，计算成本偏差(CV)。 |
| **成本估算 (Cost Estimates)** | “**大概要花多少钱**”的估算 | 对完成项目活动所需成本的量化估算。 |

### 8. 质量管理 (Quality Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **质量管理计划** | 如何保质量的“**说明书**” | 规定了项目的质量政策、标准、角色职责和活动。 |
| **质量测量指标 (Quality Metrics)** | “**用什么标准量质量**” | 详细定义了要测量什么以及如何测量，例如“缺陷密度”、“客户满意度得分”。 |
| **质量报告 (Quality Report)** | 项目的“**体检报告**” | 用于向干系人沟通项目质量管理活动的执行情况和结果。 |
| **测试与评估文件** | “**如何测，测得如何**” | 包括测试计划、测试用例和测试结果，是控制质量过程的关键产出。 |

### 9. 资源管理 (Resource Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **资源管理计划** | 如何管人管物的“**说明书**” | 规定了如何估算、获取、管理和使用实物资源与团队资源。 |
| **团队章程 (Team Charter)** | 团队的“**内部行为准则**” | 由团队共同制定，规定了团队价值观、沟通规范、决策方式等，是团队自我管理的基础。 |
| **资源分解结构 (RBS)** | “**团队与物资**”的层级图 | 按类别和类型对资源进行分解，例如按职能、按级别。 |

### 10. 沟通管理 (Communications Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **沟通管理计划** | “**和谁说，说什么，怎么说**” | 规定了项目的沟通需求、频率、格式和负责人，确保信息在正确的时间被正确的人接收。 |

### 11. 风险管理 (Risk Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **风险管理计划** | 如何管风险的“**说明书**” | 规定了如何进行风险管理活动，包括方法、角色、预算和风险类别。 |
| **风险登记册 (Risk Register)** | 项目的“**风险问题清单**” | 一个**动态**的文档，记录已识别的单个风险、其分析结果和应对计划。 |
| **风险报告 (Risk Report)** | 项目的“**整体风险总结**” | 提供关于整体项目风险以及主要单个风险的信息，用于向干系人沟通。 |

### 12. 采购管理 (Procurement Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **采购管理计划** | 如何搞采购的“**说明书**” | 规定了如何管理从“制定采购决策”到“合同收尾”的全过程。 |
| **采购策略 (Procurement Strategy)** | “**买什么，怎么买**”的决策 | 决定是自制还是外购，以及采用何种合同类型和交付方式。 |
| **招标文件 (Bid Documents)** | “**招标说明书**” (RFP, RFQ) | 向潜在卖方提供信息，以便他们提交方案和报价。 |
| **采购工作说明书 (SOW)** | 给卖方的“**详细购物清单**” | 详细说明要从外部采购的产品、服务或成果。是采购合同的重要组成部分。 |
| **协议 (Agreements)** | “**合同**” | 任何约束双方的法律文件，明确了双方的权利和义务。 |

### 13. 干系人管理 (Stakeholder Management)

| 文档名称 | “人话”解读 | 核心作用与考点 |
| :--- | :--- | :--- |
| **干系人参与计划** | 如何向上社交的“**向上管理手册**” | 制定用于促进干系人有效参与项目决策和执行的策略。 |
| **干系人登记册 (Stakeholder Register)** | 项目的“**关键人物通讯录**” | 记录了项目干系人的身份、评估信息和分类。用于规划如何与他们互动。 |
