# 07-PMP核心文档指南

> 本文档是您PMP备考的“文档说明书”。它旨在帮您彻底厘清PMP考试中那些名称相似、但用途各异的“计划”、“基准”、“登记册”、“报告”等核心文档，让您在面对情景题时，能瞬间定位到题干所指的正确文件。

---

## 整合管理 (Integration Management)

| 文档名称 | 核心作用 (Core Purpose) | “人话”解读 (Plain Language) | 常见场景 / 考点 |
| :--- | :--- | :--- | :--- |
| **项目章程** | **授权性文件**，正式启动项目，并授权项目经理使用组织资源。 | 项目的“**出生证明**”和项目经理的“**授权委托书**”。 | 项目经理被任命后，**首先**要寻找或制定的文件。没有章程，就名不正言不顺。 |
| **项目管理计划** | **综合性文件**，整合并定义了如何执行、监控和收尾项目的“总纲”。 | 项目的“**总作战地图**”，由所有子计划（如范围、进度、成本计划等）汇编而成。 | 考题问“如何做某事？”时，如果选项里有“遵循项目管理计划”，通常是高分选项。 |
| **变更日志** | **记录性文件**，用于记录所有变更请求及其处理状态。 | 项目的“**变更流水账**”，记录了所有变更的来龙去脉。 | 用于跟踪变更状态，并向干系人沟通变更处理结果。 |
| **经验教训登记册** | **知识性文件**，在项目全周期中持续记录从项目中获得的知识和教训。 | 项目的“**成长日记**”或“**错题本**”。 | 经验教训**贯穿项目始终**，而不仅仅是收尾时才做。 |

## 范围管理 (Scope Management)

| 文档名称 | 核心作用 (Core Purpose) | “人话”解读 (Plain Language) | 常见场景 / 考点 |
| :--- | :--- | :--- | :--- |
| **范围管理计划** | **指南性文件**，规定了如何定义、确认和控制范围。 | “范围工作的**游戏规则手册**”。它告诉你“如何管范围”，而不是“范围是什么”。 | 考题问“如何处理范围变更？”或“如何定义范围？”时，答案是“遵循范围管理计划”。 |
| **需求文件** | **输入性文件**，记录了所有干系人的需求。 | “来自所有人的**愿望清单**”。内容可能很庞杂，甚至相互冲突。 | 这是定义项目范围的基础和源头。 |
| **项目范围说明书** | **定义性文件**，详细描述了项目范围、主要可交付成果、假设条件和制约因素。 | “项目的**官方承诺书**”。明确定义了“我们要做什么”和“我们不做什么”。 | 判断某项工作是否在范围内时，最重要的参考文件。 |
| **WBS / WBS词典** | **分解性工具/文件**，将项目范围逐层分解为更小的工作包。 | “项目的**任务分解清单**”。WBS是图，WBS词典是图中每个任务的详细文字说明。 | WBS是范围管理的核心工具，是估算成本、制定进度的基础。最低层是“工作包”。 |
| **范围基准** | **测量性基准**，是经过批准的项目范围说明书、WBS和WBS词典的集合。 | “范围的**官方测量尺**”。所有范围相关的绩效都要对照它来衡量。 | 当发生变更时，需要拿变更请求与范围基准进行比较，以评估影响。 |

## 进度管理 (Schedule Management)

| 文档名称 | 核心作用 (Core Purpose) | “人话”解读 (Plain Language) | 常见场景 / 考点 |
| :--- | :--- | :--- | :--- |
| **进度管理计划** | **指南性文件**，规定了如何制定、管理和控制项目进度。 | “进度工作的**游戏规则手册**”。 | 规定了进度估算的单位、准确度、控制阈值等。 |
| **活动清单** | **清单性文件**，包含了为完成项目所需执行的全部进度活动。 | “项目的**详细任务列表**”，是WBS工作包的进一步分解。 | 每个活动都应有唯一的ID和清晰的描述。 |
| **里程碑清单** | **清单性文件**，列出了项目中所有重要的里程碑及其属性。 | “项目的**关键节点清单**”。里程碑是持续时间为零的重要时间点。 | 用于向高层管理人员和客户报告关键进展。 |
| **进度基准** | **测量性基准**，是经过批准的项目进度计划版本。 | “进度的**官方测量尺**”。 | 实际进度应与进度基准进行比较，以确定是否存在进度偏差(SV)。 |

---

## 成本管理 (Cost Management)

| 文档名称 | 核心作用 (Core Purpose) | “人话”解读 (Plain Language) | 常见场景 / 考点 |
| :--- | :--- | :--- | :--- |
| **成本管理计划** | **指南性文件**，规定了如何估算、预算和控制项目成本。 | “成本工作的**游戏规则手册**”。 | 规定了成本的计量单位、准确度、控制阈值等。 |
| **成本估算** | **数据性文件**，对完成项目活动所需成本的可能量化评估。 | “完成各项任务**大概要花多少钱**”。 | 估算可以是范围或量级，并应包含其依据。 |
| **成本基准** | **测量性基准**，是经过批准的、按时间段分配的项目预算。 | “成本的**官方测量尺**”，是一条按时间展开的S曲线。 | 实际支出(AC)应与成本基准进行比较，以确定是否存在成本偏差(CV)。 |

---

## 风险管理 (Risk Management)

| 文档名称 | 核心作用 (Core Purpose) | “人话”解读 (Plain Language) | 常见场景 / 考点 |
| :--- | :--- | :--- | :--- |
| **风险管理计划** | **指南性文件**，规定了如何开展项目的风险管理活动。 | “风险工作的**游戏规则手册**”。 | 定义了风险的分类、概率和影响的定义、风险偏好等。 |
| **风险登记册** | **动态清单**，记录已识别的单个项目风险的详细信息。 | “项目的**风险问题清单**”，随着项目的进行而持续更新。 | 识别出新风险后，第一步就是将其记录到风险登记册中。 |
| **风险报告** | **总结性文件**，提供关于整体项目风险以及单个项目风险信息的概述。 | “项目的**风险状况总结报告**”。 | 用于向干系人沟通项目当前的主要风险和整体风险敞口。 |

---

## 干系人管理 (Stakeholder Management)

| 文档名称 | 核心作用 (Core Purpose) | “人话”解读 (Plain Language) | 常见场景 / 考点 |
| :--- | :--- | :--- | :--- |
| **干系人登记册** | **清单性文件**，记录了项目干系人的身份、评估和分类信息。 | “项目的**关键人物通讯录**”。 | 在项目启动阶段，识别干系人后应立即创建此文件。 |
| **干系人参与计划** | **指南性文件**，识别用于促进干系人有效参与的策略和行动。 | “**搞定关键人物的行动计划**”。 | 针对权力/利益方格中不同象限的干系人，制定不同的引导策略。 |

---

## 采购管理 (Procurement Management)

| 文档名称 | 核心作用 (Core Purpose) | “人话”解读 (Plain Language) | 常见场景 / 考点 |
| :--- | :--- | :--- | :--- |
| **采购管理计划** | **指南性文件**，规定了如何进行采购管理。 | “采购活动的**游戏规则手册**”。它告诉你“怎么买”，而不是“买什么”。 | 考题问“如何选择合同类型？”或“如何评估卖方？”时，答案通常是“遵循采购管理计划”。 |
| **采购工作说明书 (SOW)** | **描述性文件**，详细说明要采购的产品、服务或成果。 | 给卖方的“**详细购物清单**”或“**需求说明书**”。 | 在决定外购后，项目经理需要准备这份文件，以便让卖方清楚地知道要做什么。 |
| **建议邀请书 (RFP) / 报价邀请书 (RFQ)** | **询价文件**，用于从潜在卖方处获取建议或报价。 | “**询价单**”。RFP用于解决方案不明确时（想听听你的建议），RFQ用于产品规格明确时（只想知道你报多少钱）。 | 当需要从多个供应商中选择一个时，会发出这些文件。 |
